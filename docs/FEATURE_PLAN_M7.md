# Feature Plan — M7: Admin Navigation Menu

## Цель

Добавить навигационное меню в хедер для пользователя с ролью `parent`.
Меню позволяет переключаться между разделами: Чат и Менеджмент пользователей.
Дети (роль `child`) меню не видят.

---

## Требования

- Меню отображается **только для `parent`**
- Пункты меню: **Чат** и **Менеджмент пользователей**
- Активный пункт подсвечивается
- После логина `parent` попадает на страницу **Чата**
- В хедере справа: имя пользователя + кнопка **Logout**
- `child` видит хедер без меню (только имя + Logout)
- Только desktop Chrome (мобильная адаптация не требуется)

---

## Архитектурное решение

Меню реализуется на фронтенде — компонент `AppHeader` уже существует.

- Роль пользователя хранится в auth-контексте (`AuthContext`) — используем её для условного рендера меню
- Навигация через `react-router-dom` (`<NavLink>` для автоматической подсветки активного пункта)
- Logout очищает токен и редиректит на `/login`

---

## Что нужно сделать

### Frontend

#### M7.1 — AppHeader: меню для Parent

**`src/components/AppHeader.jsx`:**
- Если роль `parent` — показывать два `<NavLink>`:
  - **Чат** → `/chat`
  - **Менеджмент пользователей** → `/users`
- Активный пункт получает CSS-класс `active` (через `NavLink`)
- Если роль `child` — навигационные ссылки не рендерить
- Справа: имя пользователя из auth-контекста + кнопка **Logout**

**`src/components/AppHeader.module.css`:**
- Стили хедера: flex-контейнер, меню слева, имя+logout справа
- Стиль активного пункта (подсветка)

#### M7.2 — Redirect после логина

**`src/pages/LoginPage.jsx`** (или роутинг):
- После успешного логина — редирект на `/chat` для обеих ролей

#### M7.3 — Локализация

Добавить в `ru.json` / `en.json`:
```json
"nav": {
  "chat": "Чат",
  "users": "Менеджмент пользователей",
  "logout": "Выйти"
}
```
